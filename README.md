# 医院信息管理系统

## 项目简介
基于 SSM（Spring + SpringMVC + MyBatis）框架开发，结合 Layui 前端框架实现，面向医院日常管理场景，支持管理员、医生、患者三类角色，实现医院业务流程的数字化管理。

## 技术栈
- 后端：Spring、SpringMVC、MyBatis
- 前端：JSP、HTML、CSS、JavaScript、Layui
- 数据库：MySQL 8.0
- 构建工具：Maven 3.6.3
- 服务器：Tomcat 9.0.30
- 开发环境：JDK 8、IDEA

## 系统架构
项目采用 MVC 分层架构：
- 视图层：JSP + Layui 实现页面渲染与交互
- 控制层：SpringMVC 处理请求分发与响应
- 业务层：Spring 管理业务逻辑与事务
- 持久层：MyBatis 实现数据库操作

## 核心功能模块
### 管理员模块
- 用户管理：增删改查、角色分配
- 医生管理：信息维护、科室绑定
- 患者管理：档案管理、挂号记录查询
- 科室管理：添加、编辑、删除
- 公告管理：发布与维护医院公告
- 日志管理：登录日志与操作日志记录

### 医生模块
- 个人信息管理
- 预约挂号列表查看
- 病历编写、修改与查询
- 接诊状态管理

### 患者模块
- 用户注册与登录
- 预约挂号：科室、医生、时间选择
- 个人预约记录查询
- 个人信息与密码修改

## 数据库设计
核心数据表：
- user（用户表）
- tb_doctor（医生表）
- tb_depart（科室表）
- tb_reservation（预约挂号表）
- tb_note（病历表）
- tb_promotion（公告表）
- role、menu、rolemenu（权限菜单表）

## 项目亮点
- 分层架构清晰，低耦合、易维护、易扩展
- 完善的权限控制，三类角色操作隔离
- Layui 前端界面简洁美观，交互友好
- 预约冲突校验，避免重复挂号
- 科室删除安全校验，防止误删数据
- 完整日志记录，便于问题追踪与管理

## 系统测试

- 医生、患者、科室增删改查：通过
- 预约挂号与冲突检测：通过
- 病历管理与公告发布：通过
- 分页查询与表单验证：通过
