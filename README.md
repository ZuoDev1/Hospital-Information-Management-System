# Hospital-Information-Management-System
 医院信息管理系统SSM+Layui
🚀 核心技术栈
表格
分类	技术 / 框架
后端框架	SSM（Spring + SpringMVC + MyBatis）
前端技术	JSP、HTML、CSS、JavaScript、Layui
数据库	MySQL 8.0
构建工具	Maven 3.6.3
服务器	Tomcat 9.0.30
开发环境	JDK 8、IDEA 2020.03
🧩 系统架构（MVC 分层）
视图层：JSP + Layui 页面渲染
控制层：SpringMVC 接收请求、分发响应
业务层：Spring 管理业务逻辑
持久层：MyBatis 操作 MySQL 数据库
✅ 核心功能模块
1. 管理员模块
用户管理（增删改查、角色分配）
医生管理（信息维护、科室绑定）
患者管理（档案、挂号记录）
科室管理（添加 / 编辑 / 删除）
医院公告发布与展示
登录日志记录
2. 医生模块
个人信息管理
查看预约挂号列表
病历编写、修改、查询
接诊状态管理
3. 患者模块
注册 / 登录
预约挂号（选科室 → 选医生 → 选时间）
查看个人预约记录
修改密码、个人信息
📊 数据库设计（核心表）
user（用户表：管理员 / 医生 / 患者）
tb_doctor（医生表）
tb_depart（科室表）
tb_reservation（预约挂号表）
tb_note（病历表）
tb_promotion（公告表）
role、menu、rolemenu（权限菜单）
🎯 项目亮点
MVC 分层清晰，低耦合、易维护、易扩展
权限控制完善，三角色隔离操作
Layui 前端界面简洁美观、交互友好
预约冲突校验，避免重复挂号
科室删除安全校验，防止误删
完整日志与统计，支持管理决策
🧪 系统测试结果
登录、注册、权限控制：通过
医生 / 患者 / 科室增删改查：通过
预约挂号、冲突检测：通过
病历管理、公告发布：通过
分页、查询、表单验证：通过
